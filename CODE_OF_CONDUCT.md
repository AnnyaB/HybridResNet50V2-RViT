# Code of Conduct

## Purpose

This repository supports research and educational work on pathology-guided medical image classification, shortcut-learning mitigation, explainability, uncertainty analysis, and reusable PFD-GSTE guidance modules.

This project welcomes respectful use, discussion, review, and extension of this work. The aim is to maintain a constructive environment where students, researchers, engineers, and contributors can learn from the project and build on it responsibly.

## Expected behaviour

Everyone participating in this project is expected to:

* communicate respectfully and constructively;
* give clear technical feedback without personal attacks;
* acknowledge prior work, datasets, libraries, and contributors appropriately;
* report bugs, limitations, or reproducibility issues honestly;
* respect the project’s research-only and educational purpose;
* avoid overstating the clinical capability of the models or outputs;
* use the code, trained models, and guidance modules responsibly and in line with the repository license and dataset terms.

## Unacceptable behaviour

Unacceptable behaviour includes:

* harassment, abuse, discrimination, threats, or personal attacks;
* intentionally misleading claims about model performance, clinical readiness, or safety;
* presenting this software as a certified medical device or diagnostic system;
* attempting to use the repository for clinical decision-making, patient management, or treatment decisions;
* uploading, sharing, or requesting private patient data without appropriate consent, governance, and ethical approval;
* redistributing third-party datasets in ways that violate their original terms;
* removing attribution, citation, or license notices from reused code or documentation;
* disruptive spam, malicious code, or attempts to damage the repository or users’ systems.

## Responsible research use

This project is not a clinical product. It is intended for research, education, reproducibility, and further method development.

Users should independently validate any modifications, models, or results before using them in new research. Any external dataset evaluation should clearly describe the dataset source, label mapping, preprocessing, and limitations. Results should not be reported as clinical evidence unless supported by appropriate validation, governance, and expert review.

## Reporting concerns

Concerns about behaviour, misuse, or repository safety can be reported to the repository owner through GitHub.

When reporting an issue, please include:

* a short description of the concern;
* links, screenshots, or commit references where relevant;
* whether the concern is about conduct, attribution, licensing, data use, model misuse, or another issue.

Reports will be reviewed as fairly and promptly as possible.

## Enforcement

The repository owner may take appropriate action in response to unacceptable behaviour, including:

* clarifying expectations;
* editing or removing inappropriate comments or content;
* closing issues or discussions;
* restricting participation in the repository;
* reporting severe misuse to the relevant platform or institution where appropriate.

Enforcement decisions should aim to protect respectful collaboration, academic integrity, responsible research practice, and safe use of medical AI code.

## Attribution and citation

Users who build on this work should preserve license notices and cite the project as described in `CITATION.cff` and the README. Reuse of the code is permitted under the MIT License, but proper attribution remains expected in academic and research contexts.

## Scope

This Code of Conduct applies to repository issues, pull requests, discussions, documentation, code review, and any project-related communication linked to this repository. External contributors may suggest changes through issues or pull requests, but accepted changes remain at the repository owner’s discretion.
